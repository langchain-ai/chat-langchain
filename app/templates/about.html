About

True story: I had to answer an enterprise security questionaire recently. This is not something you want to do regularly. It can be 100s of questions relating to your policies, backup procedures etc. Luckily I had really good docs, but it still took an age to look at the question then scan through several docs to get an answer. It took a couple of days. No kidding. I'm an NLP nerd so I figured; "Hey there must be a better way...why can't I give my private documents to GPT and get it to answer the questions?" 2 hours and 2 wines later....I had a basic python script that did just that. Then we build this site to showcase how to query your own documents, in this case a public set of documents, but it could really be "any documents".

So how did you build it?

Look, we like simple things. Boring things. Things that have a clear path to you know...deploying actual software.

But what technology?

Ok get ready for the Nerd Salad. Everything is written in Python. We are using FastAPI to serve the pages, websockets and Jinja2 for templating. The rest is TailwindCSS and AlpineJS. The "chat" uses the wonderful Langchain library and specifically the ConversationalRetrievalChain. An example is here.. We're using Pinecone to store the vector embeddings of the documents (which is a fancy way of saying - turning the text of the documents into numbers that Open AI understands).

Can we chat?

Sure! People that know us, know we love a chat. We're super interested to see if we can help anyone else, and if you can think of other use cases - probably with propeitary/private data. We're in Wellington, we like coffee.
